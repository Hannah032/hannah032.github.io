    KT 단자함과의 전쟁

    1 분 소요

    며칠 전부터 데스크탑의 인터넷이 이상했습니다. 최근에 쓰는 집 iptime과 독서실 iptime이 똑같아서 그런지 비밀번호를 항상 다시 치고 들어가야 했습니다. 그렇게 제 옆에만 있던 iptime 집 공유기가 원래 동생 것이어서 다시 돌아가게 되자 문제가 생겼습니다.

    여러 브라우저를 띄워놓고 빠른 인터넷을 쓰고 싶었던 저는 LAN 유선을 꽂아 노트북을 쓰려고 했습니다. 분명 유선으로 연결되어 있는데, 화면 오른쪽 아래에는 ‘인터넷 연결 안 됨’이라는 냉정한 문구가 떠 있었습니다.

    1️⃣ 원인 탐색 – 명령어와 로그

    명령 프롬프트를 열어 다음 명령어들을 입력했지만 모두 실패했습니다.

    ipconfig /release
ipconfig /renew
ping 8.8.8.8

    DNS 서버와의 연결이 끊긴 상태였고, IPv4 주소는 192.168.0.6까지 받아오지만 게이트웨이 192.168.0.1 바깥으로 나가질 못했습니다.

    이후 아래 명령어들을 차례로 실행했습니다.

    netsh winsock reset
netsh int ip reset
ipconfig /release
ipconfig /flushdns
ipconfig /renew

    이걸로 ipTIME에서 남았던 IP 주소, DNS, 네트워크 프로필이 모두 리셋되었습니다.

    그 다음에는 와이파이 프로필을 직접 지웠습니다.

    netsh wlan delete profile name="iptime"
netsh wlan delete profile name="iptime_1"
netsh wlan delete profile name="Wi-Fi 2"

    2️⃣ 단자함 조사 – 불빛 없는 포트

    혹시나 싶어 벽면 단자함을 열었습니다. 작은 허브에 파란선들이 빽빽하게 꽂혀 있었는데, 일부 포트의 불빛이 꺼져 있었습니다. 손으로 케이블을 뺐다 다시 꽂아도 여전히 불이 들어오지 않았습니다.

    결국 판단했습니다. “이건 내가 해결할 수 있는 영역이 아닐 수도 있겠구나.”

    3️⃣ 기사님 호출 → 출장비의 벽

    KT 고객센터에 전화를 걸었습니다. 하지만 상담원께서 말하길, 기사님 방문에는 출장비가 발생한다는 겁니다. 잠시 고민하다가 취소했습니다. “출장비 내느니, 다른 방법을 찾아보자.”

    4️⃣ 무선 랜카드의 귀환

    예전에 쿠팡에서 사두었던 무선 랜카드(USB형)가 떠올랐습니다. 침착하게 가방에서 꺼내어 KT망의 와이파이를 연결하고 무리한 작업을 그만두었습니다.

    제약이 있지만 연결 버튼을 누르고 잠시 기다리니, 드디어 인터넷이 살아났습니다. 랜선을 통한 유선 연결은 여전히 고장 났지만, 무선으로 응급처치에 성공했습니다.

    5️⃣ 기록의 의미

    이 글은 단순한 장애 기록이 아닙니다. 문제가 생겼을 때 원인을 탐색하고, 대체 경로를 찾아내는 과정 그 자체에 대한 기록입니다. 프로그래밍에서 에러를 만날 때처럼, 인터넷도 결국은 연결의 문제였습니다.

    에러는 피하는 게 아니라, 관찰하고 기록하고 수정하는 것입니다. 그게 결국 연결을 복구하는 과정이니까요.

    오늘의 교훈

    “인터넷이 안 될 땐, 복잡한 원인보다 눈앞의 연결부터 점검하자.”
